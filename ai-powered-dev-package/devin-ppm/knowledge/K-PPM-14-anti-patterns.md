# K-14: PPM Anti-patterns

**Trigger:** When reviewing sprint processes, when something feels wrong in the workflow, or when debugging why an iteration went badly.

**Content:**

| Anti-pattern | Why Bad | Correct Approach |
|-------------|---------|-----------------|
| Skipping Assessment | Plan based on stale info | Always read status before planning |
| Planning without grooming | Sprint has irrelevant/oversized tasks | Refine backlog before selecting |
| Expanding scope mid-sprint | Sprint never finishes, velocity unmeasurable | Add discoveries to backlog, not current sprint |
| Implementing without a requirement | No traceability, no test criteria, no DoD | Create the requirement first, even if minimal |
| Skipping test plan | Don't know what "done" means | Every task needs testable ACs before starting |
| Not closing ACs in requirement files | Requirements show incomplete despite progress | Phase 5.2 is mandatory |
| Deferring without reason | Backlog becomes graveyard | Delete or defer with specific revisit date |
| Gold-plating | Doing more than AC wastes capacity | Implement exactly the AC |
| Skipping retrospective | Same mistakes repeat | Even 3 bullet points is valuable |
| Updating docs only at sprint close | Mid-sprint status opaque | Update task status as you complete each task |
