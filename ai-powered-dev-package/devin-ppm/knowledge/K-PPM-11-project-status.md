# K-11: Project Status & Maturity

**Trigger:** When updating project-status.md, when assessing component health, or when deciding if a component is production-ready.

**Content:**

`status/project-status.md` contains:
- **Component Health** table: component, maturity, quality, notes
- **What's Built:** summary of completed capabilities (reference sprints)
- **Known Gaps:** categorized list with backlog task references
- **Half-done Items:** started but incomplete, with remaining work

**Maturity levels:**
- **Prototype:** Proof of concept, not for production
- **Alpha:** Core functionality works, gaps remain, no SLA
- **Beta:** Feature-complete for V1 scope, some gaps, partial SLA
- **Production-ready:** SLAs met, tested, monitored, documented

**testing/coverage.md** contains:
- Summary table: module, unit %, integration tests, E2E flows
- Coverage gaps linked to backlog tasks
